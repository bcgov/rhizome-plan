How it Works

A knowledge graph connects to data sources across government, enriches the data by finding connections across all of them, and creates human- and machine-understandable output.
Data is accessed in different ways - Rhizome meets the data where it is, whether it's in a SQL database, locked behind a legacy system, or in a prehistoric format. Rather than forcing integration through rigid access patterns, Rhizome adapts.
Missed graphy theory in college? Dig lets you explore the data fabric of government without specialized knowledge. Identify paths between data points. Dig traverses virtualized data, finding patterns across disconnected data stores.
Rhizome captures critical meta-data like quality measures, access constraints, authority, provenance - it's all part of the knowledge graph. Identify rival stores, flag conflicting or redundant data. Set constraints on data flows.
Connect data no matter the format, structure, or location across silos and lines of business.
End-to-end, Rhizome can be built using robust, mature open source tools and frameworks. Key elements will need to be custom built - but the heavy lifting can be done with Apache Foundation systems like Airflow, Spark, Nifi, Atlas and Camel.
Knowledge graph technology has been used successfully for over a decade in the private sector. SPARQL, SHACL, and GraphQL are common standards. Rhizome is built upon RDF and OWL - both W3C Open standaards.
